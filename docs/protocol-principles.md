# Protocol Principles

THANK should remain boring, transparent, and useful.

## Launch Boundaries

- No ICO
- No pre-mine
- No founder allocation
- No price predictions
- No guaranteed returns
- No referral rewards
- No exchange-listing hype

## Technical Principles

- Use existing payment assets first.
- Make manifests human-readable.
- Make verification status obvious.
- Prefer small auditable contracts.
- Keep receipts symbolic and non-transferable.
- Route money to maintainers and upstream dependencies before protocol fees.

## Governance Direction

Start with a transparent maintainer multisig, then move toward an advisory board or foundation-like stewardship model. Avoid pure token voting.
